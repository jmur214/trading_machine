# Trading Machine Development Summary — November 9, 2025

## 🧠 Project Overview

**Goal:**  
To develop an **autonomous trading machine** that learns, tests, and evolves its own alpha-generating strategies (“edges”) through continuous feedback, adaptive configuration, and backtesting.  

This system is designed to:
- Collect raw signals from a variety of technical or cross-sectional edges.  
- Normalize and evaluate those signals through the **Alpha Engine**.  
- Feed decisions into a **risk engine** and **execution simulator**.  
- Analyze outcomes and **improve automatically** via the **Governor feedback loop**.  

The long-term goal is a **self-learning quant system** that can autonomously test, select, and tune edges for consistent profitability.

---

## 🧩 Core Architecture

### Key Components

| Module | Description |
|--------|-------------|
| **AlphaEngine** | Central orchestrator. Collects, normalizes, aggregates, and thresholds signals from all active edges. Produces actionable trading signals. |
| **SignalCollector** | Interface between AlphaEngine and edges. Collects edge outputs (dicts or Signal objects), validates format, normalizes tickers, and merges signals for processing. |
| **Edges** | Independent alpha-generating strategies. Each encapsulates a trading hypothesis (e.g., RSI Mean Reversion, Cross-Sectional Mean Reversion). |
| **EdgeRegistry / EdgeSpec** | Maintains metadata for all active edges — IDs, parameters, versions, categories. Allows dynamic discovery and version management. |
| **RiskEngine** | Determines position sizing, leverage, stop-loss, and take-profit logic based on available equity and portfolio exposure. |
| **ExecutionSimulator** | Mimics live fills during backtests, accounting for latency, slippage, and commissions. Produces realistic fill records for analysis. |
| **Governor** | Feedback controller that updates thresholds, confidence weighting, and edge participation based on performance metrics. |
| **run_backtest.py** | CLI entry point that initializes the engine stack (AlphaEngine → RiskEngine → ExecutionSimulator → Governor) and runs time-series simulations. |
| **DataManager** | Abstract layer for input normalization. Converts data from sources (Yahoo, Alpaca, Binance, CSV) into uniform OHLCV DataFrames for edge processing. |

## 🛠️ Major Changes Since the GitHub Agent Review

### ✅ 1. RSI Mean Reversion (`rsi_mean_reversion.py`)
**Purpose:** Identify short-term overbought/oversold conditions relative to a long-term mean.

#### Key Upgrades
- Rewritten from scratch for modularity and debug visibility.  
- Introduced parameters:
  - `rsi_window` (default 14)
  - `ma_window` (default 200)
  - `rsi_lower` (default 45.0)
  - `rsi_upper` (default 55.0)
  - `confidence_base` (default 0.7)
  - `min_price` filter (default 5.0)
- Implemented `_confidence()` model that adjusts signal strength based on RSI deviation from midpoint.
- Added `_build_meta()` for human-readable dashboards and logs:
RSI(14)=63.4, MA(200)=215.56, Price=220.03
- Supports **long**, **short**, and **exit** signals:
- Long when RSI < lower band and price > MA.  
- Short when RSI > upper band and price < MA.  
- Exit when RSI > upper band and price > MA.  
- Integrated full error handling and debug traces: [EDGE][DEBUG][rsi_mean_reversion_v1] AAPL: rsi_now=63.30, ma_now=157.23, price_now=193.17
                                                    [EDGE][DEBUG][rsi_mean_reversion_v1] Generated 3 signals ([‘exit’, ‘exit’, ‘long’])
#### Dashboard & Registry Integration
- Signals include `meta["explain"]` and `meta["edges_triggered"]` for visualization.
