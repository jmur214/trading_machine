# Core File Index
This is a high-level, quick-reference guide to the primary components of the Trading Machine. For a deeply specific, file-by-file breakdown, refer to `extensive_files.md`.

## 1. Analytics & Intelligence (`analytics/`, `intelligence/`)
Responsible for post-trade feedback loops (updating the Governor) and processing external data like news sentiment.

## 2. Backtester (`backtester/`)
The core simulation loop. Connects all engines, handles historical iteration, realistically simulates fills (slippage/commissions), and maintains portfolio equity over time.

## 3. Brokers (`brokers/`)
The execution layer interfacing with external platforms (e.g., Alpaca for Paper/Live) or internal simulated environments.

## 4. Cockpit Dashboard (`cockpit/`)
The web-based visual analytics terminal. Includes logging schemas, real-time reactive callbacks, and front-end tabs for performance, governor state, and system control. Keep in mind, **cockpit/dashboard/ is obsolete.**, we are now using *tabs & callbacks* for the UI.

## 5. Engines (`engines/`)
The 4 pillars of the trading logic:
- **Engine A (Alpha)**: Generates trade signals to find "edges" in the market.
- **Engine B (Risk)**: Sizes positions and places orders.
- **Engine C (Portfolio)**: Manages holistic portfolio balance and sleeves.
- **Engine D (Governor)**: Adaptively reweights strategies based on performance.

## 6. Data (`data/`)
Stores historical trade logs, portfolio snapshots, configuration weights, and persistent metrics generated by the system.
