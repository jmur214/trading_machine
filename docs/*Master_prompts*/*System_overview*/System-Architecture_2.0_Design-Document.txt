1 Objectives (What v2 achieves)
	•	Portfolio-first: Shift from “trade runner” to portfolio manager with sleeves, target weights, drift control, and rebalancing.
	•	Unified OMS: Same order/position/fill model for backtest, paper, (future) live.
	•	Adaptive allocation: The Governor produces edge weights that actually control capital (not just get logged).
	•	Robust data: Replace brittle CSVs with DuckDB/Parquet + run IDs and schemas.
	•	Explainable: Persist reasons/metrics for weight changes; dashboards show “why.”

2 Operating Modes
	•	Backtest (historical bars → fills → portfolio → logs)
	•	Paper (Alpaca REST/WS → unified OMS → same logs/UI)
	•	Live (future) (same OMS, broker adapter swaps)

All modes use the same Order → Fill → Position → Snapshot contracts.

3 Core Components & Responsibilities

A. Portfolio Engine (new)

Role: Single source of truth for positions, cash, equity, PnL, target weights, and rebalancing.

Key responsibilities
	•	Maintain positions (avg price, qty, side, unrealized PnL).
	•	Apply fills to update cash/equity.
	•	Track sleeves (e.g., core_holdings, tactical_edges, experimental).
	•	Enforce target allocations by sleeve/edge; compute drift and generate rebalance orders.
	•	Publish snapshots (equity, exposures, realized/unrealized PnL).
	•	Provide risk hooks for limits (beta/exposure/turnover/day-loss).

Interface (Python)
class PortfolioEngine:
    def __init__(self, starting_cash: float, run_id: str, schema_store, risk_policy):
        ...

    def apply_fill(self, fill: "Fill") -> None: ...
    def current_positions(self) -> dict[str, "Position"]: ...
    def snapshot(self, ts: pd.Timestamp) -> dict: ...
    def set_targets(self, targets: dict[str, float]) -> None:  # edge_id -> weight (0..1)
    def compute_rebalance_orders(self, prices: dict[str, float]) -> list["Order"]: ...
    def pnl_summary(self) -> dict: ...

B. Unified OMS (Order Management System) (new)

Role: Normalize orders → fills across backtest, paper, live.

Key responsibilities
	•	Standard datamodels: Order, Fill, Position.
	•	Execution adapters:
	•	SimExecutionAdapter (wraps current ExecutionSimulator)
	•	AlpacaExecutionAdapter (wraps AlpacaBroker)
	•	Route all fills through CockpitLogger (or DB writer).

Datamodels
@dataclass
class Order:
    ts: pd.Timestamp
    symbol: str
    side: Literal["buy","sell","short","cover"]
    qty: float
    type: Literal["market","limit","stop","stop_limit","trailing"] = "market"
    limit_price: float | None = None
    stop_price: float | None = None
    tif: Literal["day","gtc"] = "gtc"
    edge_id: str | None = None
    sleeve: str | None = None
    meta: dict = field(default_factory=dict)

@dataclass
class Fill:
    ts: pd.Timestamp
    symbol: str
    side: str
    qty: float
    price: float
    commission: float = 0.0
    order_id: str | None = None
    edge_id: str | None = None
    meta: dict = field(default_factory=dict)

@dataclass
class Position:
    symbol: str
    qty: float
    avg_price: float
    unrealized_pnl: float = 0.0
    meta: dict = field(default_factory=dict)

Adapter interface
class ExecutionAdapter:
    def submit_orders(self, orders: list[Order]) -> list[Fill]: ...
    def close_position(self, symbol: str) -> list[Fill]: ...

C. Governor (evolved)

Role: Turn performance into target allocation by edge (and sleeves); save explainable rationales.

Key responsibilities
	•	Read per-edge metrics (Sharpe, MDD, trade count, corr).
	•	Compute weights using policy (e.g., SR-weighted softmax with caps & decay).
	•	Persist edge_weights, weight_history, metrics, rationales.
	•	Optional diversification penalty if edges are highly correlated.
	•	Optional sleeve policy: e.g., core_holdings min 40%, tactical_edges 50%, experimental 10%.

Policy example
w_i = softmax(k * SR_i) with bounds: min_w <= w_i <= max_w
Apply decorrelation: w_i = w_i * (1 - λ * avg_corr_i)
Renormalize within sleeve, then across sleeves.

D. Risk Engine 2.0 (upgraded)

Role: Enforce portfolio-level risk limits and turn signals into orders respecting caps.

Controls
	•	Vol-targeting (scale orders so ex-ante vol ≈ target).
	•	Exposure caps (gross, net, per-sector, per-asset).
	•	Turnover limits.
	•	Drawdown brakes (pause trading beyond threshold).
	•	Beta caps (vs benchmark).

Interface
class RiskEngine:
    def size_orders(self, signals: list[dict], portfolio: PortfolioEngine, prices: dict) -> list[Order]: ...
    def enforce_limits(self, orders: list[Order], portfolio: PortfolioEngine) -> list[Order]: ...

E. Data Layer (DuckDB/Parquet + Run IDs)

Role: Replace CSV-only with structured storage. Keep CSV writes for compatibility during migration.

Artifacts
	•	runs (run_id, mode, started_at, ended_at, config_hash)
	•	trades (run_id, ts, symbol, side, qty, price, commission, edge_id, sleeve, realized_pnl, meta)
	•	snapshots (run_id, ts, equity, cash, mv, realized, unrealized, exposure, positions_json)
	•	weights (run_id, ts, edge_id, weight, sleeve, rationale_json)
	•	metrics (run_id, scope={portfolio|edge_id}, sharpe, mdd, vol, cagr, winrate, period)

Data access
class DataStore:
    def __init__(self, db_path="data/trading.duckdb"): ...
    def begin_run(self, mode:str, config:dict) -> str: ...
    def write_trades(self, run_id:str, fills:list[Fill]) -> None: ...
    def write_snapshot(self, run_id:str, snap:dict) -> None: ...
    def write_weights(self, run_id:str, weights:dict, rationale:dict) -> None: ...
    def close_run(self, run_id:str) -> None: ...
Migration: keep current CSV logging for a few sprints; mirror writes into DuckDB; update DataManager to prefer DB when present.

4 Control Flows

Backtest cycle
	1.	Load bars & config → run_id = datastore.begin_run("backtest", config).
	2.	Alpha → Signals (with edge_id).
	3.	Risk → Orders (respect limits).
	4.	OMS (SimAdapter) → Fills at next open (slippage/commission).
	5.	Portfolio → apply fills; snapshot each bar.
	6.	Metrics → per-edge & portfolio.
	7.	Governor → compute weights & rationale; persist.
	8.	Dashboard reads from DB (fallback CSV).
	9.	datastore.close_run(run_id).

Paper cycle
	1.	run_id = datastore.begin_run("paper", config).
	2.	Stream prices/orders from Alpaca (WS/REST).
	3.	OMS (AlpacaAdapter) → submit orders, receive fills.
	4.	Portfolio + snapshots as above; persist to DB.
	5.	Metrics/Governor nightly; dashboard live.
	6.	Close run when session ends.
