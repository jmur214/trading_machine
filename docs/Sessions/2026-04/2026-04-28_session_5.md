## What was decided

- **Realistic cost model is the new default.** Backtest_settings.json
  ships with `slippage_model: "realistic"`. Legacy 10 bps flat is
  available via `slippage_model: "fixed"` for A/B comparison.

- **Strongest-of-three is the new default benchmark gate.** Discovery's
  Gate 1 now requires the candidate to beat `max(SPY, QQQ, 60/40)
  Sharpe - margin`. Backward compat via `mode='spy_only'`.

- **Factor decomposition gate (Gate 6) added to the gauntlet.**
  Candidates need t-stat > 2 AND alpha annualized > 2% on the
  FF5+Momentum regression to pass. Skip semantics on missing cache so
  the gate doesn't block the loop when the diagnostic hasn't been run.

- **Meta-learner build deferred for explicit approval.** Four
  decisions need the user's sign-off before build: (a) sklearn vs
  xgboost first, (b) forward-return horizon, (c) tier classifications,
  (d) held-out fold size. Design doc captures these.

- **No new dependencies added.** Phase 0/1 work stays within the existing
  stack. xgboost/lightgbm/statsmodels/pandas-datareader were not
  installed — the meta-learner design proposes sklearn (already
  available) for the first build.

## What was learned

- **The factor diagnostic produced a clean signal with real and false
  alpha clearly differentiated.** 2 edges (`volume_anomaly_v1`,
  `herding_v1`) are real alpha (t > 4); 2 are marginal (t ≈ 1); 3 are
  factor beta (t < 1); 2 are quantitatively destroying value (t < -2,
  the same paused edges the lifecycle had already flagged). This means
  the v2 doc's combiner-as-bottleneck argument is valid — there ARE
  real signals to combine, not just factor exposure.

- **The realistic-cost model dramatically reduced reported slippage
  costs but isn't expected to lift Sharpe much.** The earlier diagnostic
  showed -77% cost reduction across 3 historical runs. Per the
  cross-run analysis, the legacy 10 bps flat was overcharging the
  mostly-mega-cap universe. Reported Sharpe under realistic costs is
  *modestly higher*, not lower as v2 anticipated.

- **The bare-except discipline established in earlier sessions paid
  off.** Gate 6's exception handling re-raises `TypeError` and
  `AttributeError` so future schema-drift surfaces immediately, while
  legitimate `FileNotFoundError` (cache missing) gracefully degrades
  to skip. Same pattern as Gate 3 / Gate 5.

- **Module extraction is the right pattern when a script's logic gets
  reused as a gate.** The factor-regression code now lives in
  `core/factor_decomposition.py`; the diagnostic script and Gate 6 both
  import from it. Tests for the module cover both callers.
