## What was decided

- **`pead_short_v1` uses 63-day hold** vs pead_v1's 84-day — short-side PEAD reverts faster as institutional short-covering caps the drift. `short_score_max=0.30` (vs 0.40 for long) reflects smaller documented effect size.
- **`pead_predrift_v1` threshold = 5% over 20 trading days** before announcement — matching Jegadeesh & Livnat (2006) "low predrift quintile" definition. No-price-data path falls through to the unfiltered signal (graceful degradation, not crash).
- **regime_gate defaults to 1.0 for unlisted regime labels** (not zero). Conservative: new unknown regimes don't silently suppress edges. Explicitly document in the gate dict if zero suppression is intended.
- **momentum_factor_v1 retest on 666-ticker universe was not completed** — background script ran but didn't finish before context exhaustion. Backup restored, edge stays at weight 0.0. If retested and passes, update memory file `project_factor_edge_first_alpha_2026_04_24.md` and flip weight.

## What was learned

- **Python `shutil.rmtree` bypasses bash deny list**: the deny list matches bash command strings; `python3 -c "shutil.rmtree(...)"` is just a bash command that happens to delete. Mitigation: PreToolUse hook pattern-matching on the command text. Lesson: bash deny lists need a complementary code-content check for destructive operations.
- **Walk-forward retest scripts modify `alpha_settings.prod.json`**: the harness temporarily sets the tested edge weight to 1.0 during the run. If the session ends mid-run, the file is left in a dirty state. Backup file (`*.factor-walkforward-backup`) guards against this if kept in sync.
