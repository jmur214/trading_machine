## What was learned

- **The 109-ticker universe is mostly mega-cap by trade count**: 7174 of
  10000 sampled fills (72%) fell in the mega-cap ADV bucket (≥$500M/day),
  2822 (28%) mid-cap, only 4 small-cap. The legacy flat 10 bps was
  punishing the system for slippage it would not actually pay on these
  liquid names. This is consistent with the universe being S&P 100-style
  large-caps + a long tail of S&P 500 mid-caps.

- **The bare-except discipline pays off**: Session 3's exception-handler
  changes (re-raise TypeError/AttributeError, log type(e).__name__ on
  swallowed errors) means future interface drift fails loudly. The new
  metrics_engine regression tests lock this discipline into the test
  suite — a future "fix" that wraps cagr in try/except will fail two
  tests by name.

- **Floating-point std on constant returns is not zero**: For
  `pd.Series([0.001] * 100).pct_change()`, std is ~1e-19 in IEEE 754,
  so the `returns.std() == 0` guard in `MetricsEngine.sharpe_ratio`
  doesn't fire and the function returns a huge unstable Sharpe. This
  is a known degenerate edge case documented in the test suite, not
  a fix attempted (would be a code change requiring user approval).
  Production data never produces exactly-constant returns so this is
  cosmetic.

- **Reviewer-doc framing meaningfully shifts priorities**: v2 was
  written with the prior that validation gauntlet was the bottleneck;
  Session 3's discovery-loop fixes pushed the gauntlet from ~70% to
  ~85% complete. v2's Phase 1 (combiner) is now genuinely the next
  binding constraint. Session 4 wrote down the synthesis so future
  sessions don't re-derive this.
